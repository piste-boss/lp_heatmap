## Step 2: コンバージョン計測の設定

### 2-1. Google広告側のコンバージョン設定

1. Google広告管理画面 →「目標」→「コンバージョン」→「+ 新しいコンバージョンアクション」
2. 「ウェブサイト」を選択

|---------|-----|

| コンバージョン名 | `LINE友だち追加クリック` |
| 値 | 各コンバージョンに同一の価値を割り当て → `1,000円` |
| カウント方法 | 初回のみ |
| クリックスルーCV期間 | 30日 |

#### コンバージョンアクションB: 資料請求メールクリック（セカンダリCTA設置後）

| 設定項目 | 値 |
|---------|-----|
| カテゴリ | リード > 問い合わせ |
| コンバージョン名 | `資料請求クリック` |

| カウント方法 | 初回のみ |

### 2-2. Googleタグの設置

Google広告が生成するグローバルサイトタグを `index.html` の `<head>` 内に追加する。

```html
<!-- Google Ads Global Site Tag — Clarity の直後に追加 -->
<script async src="https://www.googletagmanager.com/gtag/js?id=AW-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'AW-XXXXXXXXXX');
</script>
```
※ `AW-XXXXXXXXXX` は実際のコンバージョンIDに置き換え

### 2-3. LINE CTAクリックのイベント計測

LINE CTAボタンのクリックをコンバージョンとして計測するため、以下のいずれかで実装する。

**方法A: onclick属性を追加（簡易）**

全ての `.cta-line` リンク（`lin.ee` へのリンク）に追加:

```html
<a href="https://lin.ee/pmnqCCA" target="_blank" rel="noopener" class="cta-line"
   onclick="gtag('event', 'conversion', {'send_to': 'AW-XXXXXXXXXX/YYYYYY'});">
```

**方法B: Google Tag Manager経由（推奨・将来的）**

GTMコンテナを設置し、「クリックURL が lin.ee を含む」トリガーでコンバージョンタグを発火。

### 2-4. UTMパラメータの設定

Google広告からのLP流入を Clarity / GA で識別するため、広告のリンク先URLにUTMを付与する。

```
https://review-gpt-lp.netlify.app/?utm_source=google&utm_medium=cpc&utm_campaign={campaignid}&utm_term={keyword}&utm_content={creative}
```

Google広告の「最終ページURL のサフィックス」に以下を設定:
```
utm_source=google&utm_medium=cpc&utm_campaign={campaignid}&utm_term={keyword}&utm_content={creative}
```

## Step 6: 予算・入札戦略の設定

### 6-1. 初期予算（テストフェーズ: Week 1-4）

| 項目 | 設定値 |
|------|--------|
| 日予算 | **500円/日** |
| 月間上限目安 | **約15,000円/月** |
| テスト期間 | **最低2週間**（判断に必要なデータ量: 50クリック以上） |

### 6-2. 入札戦略

| フェーズ | 入札戦略 | 理由 |
|---------|---------|------|
| Week 1-4 | **手動CPC**（上限CPC: 200円） | 500円/日で確実に2〜3クリック/日確保するため。自動入札は予算が少ないと安定しない |
| CV 15件以上蓄積後 | **コンバージョン数の最大化** | 十分なCVデータが溜まってから自動入札に移行 |
| 安定期 | **目標CPA** | 目標CPA 3,000〜5,000円 を設定 |

> **注意**: 日予算500円で「クリック数の最大化」（自動入札）にすると、Googleが1クリックに300〜500円使い切るケースがある。手動CPCで上限を抑え、1日に複数クリックを確保する方が学習データが集まりやすい。

### 6-3. 予算感の目安（500円/日の場合）

```
想定CPC: 100〜200円（手動CPC上限200円）
日予算 500円 → 1日あたり 2〜5クリック
月間: 60〜150クリック
想定CVR 3〜5% → 月間CV: 2〜7件
想定CPA: 2,000〜7,500円/件

2週間テスト: 約7,000円で30〜70クリック分のデータ
  → スクロール深度・CTR・CVRの判断に十分
```

### 6-4. 予算500円/日を最大化するコツ

1. **広告グループは最初1つだけ稼働**: 3グループに500円を分散させるとデータが薄まる。「口コミ対策_一般」のみ有効にし、他2つは一時停止
2. **配信スケジュール**: 店舗経営者が検索しやすい時間帯に集中
   - 平日 9:00〜12:00、18:00〜23:00（営業時間外に情報収集する傾向）
   - 日曜は配信OFF（検索ボリューム少）
3. **デバイス**: モバイル入札を+20%、PC据え置き（経営者はスマホ検索が多い）
4. **2週間後に判断**: スクロール深度40%以上なら予算増額、20%以下ならKW・広告文を見直し

## Step 7: 地域ターゲティング

### 7-1. 初期設定（テストフェーズ: 500円/日）

| 設定項目 | 値 |
|---------|-----|
| ターゲット地域 | **愛知県全域** |
| 地域オプション | **「ターゲット地域にいるユーザー」**を選択 |

> **重要**: 「ターゲット地域に関心を示しているユーザー」は必ず外す。これをONにすると、愛知県外のユーザーにも表示され、500円/日の予算が無関係なクリックに消費される。

### 7-2. なぜ「刈谷駅半径3km」ではなく「愛知県全域」か

```
刈谷駅 半径3km:
  人口: 約5〜8万人
  店舗経営者: 500〜2,400人
  「口コミ 増やす」を検索する人/日: 0〜1人
  → 広告が表示すらされない日が大半。テストとして機能しない。

愛知県全域:
  人口: 約750万人
  店舗経営者: 数万人規模
  「口コミ 増やす」を検索する人/日: 数十人
  → 500円/日でも毎日2〜5クリック確保可能。2週間で判断できるデータが集まる。
```

### 7-3. 拡大フェーズ（効果確認後）

```
Phase 1: 愛知県 × 500円/日（2〜4週間テスト）
Phase 2: 東海3県（愛知・岐阜・三重）× 1,000円/日
Phase 3: 関東 or 関西の主要都市 × 2,000円/日
Phase 4: 全国 × 予算は効果に応じて調整
```

### 7-4. 地域の除外

特に不要な地域はないが、効果測定で特定地域のCVRが極端に低い場合は除外を検討。

2. 広告の審査が通るまで最大24時間待つ（通常は数時間）

### 9-2. 初日〜3日目のチェック項目

| チェック項目 | 確認場所 | 期待値 |
|------------|---------|--------|
| インプレッション | Google広告管理画面 | 1日100以上 |
| クリック数 | Google広告管理画面 | 1日5以上 |
| CTR | Google広告管理画面 | 3%以上 |
| 平均CPC | Google広告管理画面 | 300円以下 |
| LP表示 | Clarity | Google広告流入が計測されている |
| スクロール深度 | Clarity ヒートマップ | **40%以上（最重要KPI）** |
| LP滞在時間 | Clarity | 30秒以上 |

### 9-3. 初期に起こりうる問題と対応

| 問題 | 原因 | 対応 |
|------|------|------|
| インプレッションが出ない | キーワードの検索ボリュームが少ない / 入札が低い | 部分一致KWを追加 or 上限CPCを引き上げ |

| クリック後すぐ離脱 | 検索意図とLPの不一致 | 検索クエリレポートを確認し、不一致KWを除外 |
| CVが出ない | CTAが見つけにくい / LPの説得力不足 | Clarityのセッション録画で離脱ポイントを特定 |

```
   → 意図しないクエリがあれば除外KWに追加
   → CVにつながったクエリがあればキーワードに追加

2. 広告文のパフォーマンス
   → CTRが低い見出し/説明文を差し替え
   → 「最良」評価の見出しを他の広告グループにも展開

3. デバイス別パフォーマンス
   → モバイル vs PC でCVRに大きな差があれば入札調整

4. 地域別パフォーマンス
   → CV件数が多い地域に予算を集中

5. Clarity ヒートマップ
   → Google広告流入のスクロール深度・クリック箇所を確認
   → Instagram流入との比較データを蓄積
```
