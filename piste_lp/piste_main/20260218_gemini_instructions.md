# Gemini向け指示: Piste_LP Clarityデータ収集（2026/02/18）

## 目的

Microsoft ClarityからPiste_LPの1週間分のヒートマップデータを収集し、スクリーンショットを保存する。

## 保存先ディレクトリ

```
/Users/ishikawasuguru/lp_heatmap/piste_lp/piste_main/20260218_analysis/
```

このディレクトリを作成し、すべてのスクリーンショットをここに保存すること。

## 手順

### ステップ1: Clarityにアクセス

1. Microsoft Clarity（https://clarity.microsoft.com）にログイン
2. プロジェクト一覧から **「Piste_LP」** を選択

### ステップ2: 期間設定

- フィルターの期間を **「過去7日間」** に設定（2/11〜2/18をカバー）
- URLフィルターがある場合は piste_lp のURLに一致するよう設定

### ステップ3: ダッシュボード概要のスクリーンショット

ダッシュボードで以下の数値が見える状態でスクリーンショットを取得:

- **セッション数**（総数）
- **平均スクロール率**（%）
- **平均滞在時間**（秒）
- **デッドクリック率**（%）
- **クイックバック率**（%）

保存ファイル名: `dashboard_overview.png`

### ステップ4: ヒートマップデータ取得

Clarityの「ヒートマップ」タブに移動し、以下3種類のヒートマップを取得する。
すべて **モバイル表示** で取得すること。

#### 4-1. スクロールヒートマップ

1. ヒートマップ画面で「スクロール」タブを選択
2. 各地点のスクロール到達率（%）が見える状態でスクリーンショット
3. 特に以下を確認:
   - 0-5%地点の離脱率（改修前: 34.29%）
   - 50%到達率
   - ページ全体のスクロール分布

保存ファイル名: `mobile_scroll_heatmap.png`

#### 4-2. タップ/クリックヒートマップ

1. 「タップ」タブを選択
2. クリック数とクリック箇所が見える状態でスクリーンショット
3. 特に確認:
   - CTAボタン「無料体験を予約する」のタップ数
   - FAQセクションのタップ数
   - ハンバーガーメニューのタップ数
   - デッドクリックの有無

保存ファイル名: `mobile_tap_heatmap.png`

#### 4-3. アテンションヒートマップ

1. 「注意」タブを選択
2. 各セクションの滞在時間・アテンション分布が見えるスクリーンショット

保存ファイル名: `mobile_attention_heatmap.png`

### ステップ5: セッション録画

「レコーディング」タブに移動し、以下の条件でセッション録画を確認:

1. **最も滞在時間が長いセッション** を1件選び、スクリーンショットを保存
2. **典型的な離脱パターンのセッション** を1件選び、スクリーンショットを保存
3. 各セッションについて以下を記録:
   - 滞在時間
   - スクロール到達地点
   - クリックした要素
   - 離脱ポイント

保存ファイル名:
- `session_recording_1.png`
- `session_recording_2.png`

### ステップ6: デッドクリックレポート（可能であれば）

ダッシュボードの「デッドクリック」セクションのスクリーンショットも取得。

保存ファイル名: `dead_clicks.png`

## 注意事項

- すべてのスクリーンショットは **数値やラベルが読める解像度** で保存すること
- ヒートマップは **ページ全体が映る** ように、必要に応じて複数枚に分けてもOK
- 期間フィルターが「過去7日間」になっていることを必ず確認してからスクリーンショットを取得すること
- データ取得後、特筆すべき数値の変化があればテキストでもメモを残すこと
